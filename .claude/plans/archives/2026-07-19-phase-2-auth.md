## Decisions / deviations (decided up front — do not re-litigate)

1. **Refresh tokens stored raw** (parity — source stores the raw JWT; `sessions.refresh_token` is `UNIQUE` and looked up by raw value). Hashing-at-rest is deferred (open question #3), not this phase.
2. **Refresh token claims add `jti` (random UUID) + `iat`.** The source signs `{sub}` with *no* `exp` and no nonce; with a deterministic HS256 signer that risks **identical token strings** for the same user, which violates the `UNIQUE(refresh_token)` constraint. Adding a random `jti` guarantees uniqueness. Tokens are opaque to clients, so the external contract is unchanged. **Do NOT add `exp` to the refresh token** — DB `expires_at` is the sole authority for expiry (drives `REFRESH_TOKEN_EXPIRED`), exactly as in source.
3. **Access-token blacklist TTL stays hard-coded at 15 min (900 s)** to match the contract and `CLAUDE.md` (`blacklist:<accessToken>` EX 900). Config-driven TTL (open question #5) is deferred.
4. **bcrypt input truncated to 72 bytes** before hash/compare. `bcryptjs` (source) silently truncates; Go's `x/crypto/bcrypt` returns an error for >72 bytes, which would turn a >72-char password into a 500 instead of parity behavior. Truncate to the first 72 bytes in both `register` and `login` to match source. (Password min-length is 8; this only affects the long tail.)
5. **No `requireAuth`/`requireOrg`/`requirePermission` middleware this phase.** All four `/auth` routes are **public**. `logout` reads the `Authorization` header directly (no guard) to blacklist. Guards are Phase 3.

### Step 2 — Redis auth helpers (`internal/infra/redis/`)

Add a new file `internal/infra/redis/auth.go` with an `Auth` type wrapping `*redis.Client`, mirroring `RedisAuth` in the source **exactly** (key names, TTL 900 s, INCR/EXPIRE semantics):

```go
type Auth struct{ client *redis.Client }

func NewAuth(client *redis.Client) *Auth { return &Auth{client: client} }

// blacklist:<token> = "1", EX ttlSeconds
func (a *Auth) BlacklistToken(ctx context.Context, token string, ttl time.Duration) error
func (a *Auth) IsBlacklisted(ctx context.Context, token string) (bool, error) // EXISTS == 1

// login:attempts:<email>
func (a *Auth) IncrementLoginAttempts(ctx context.Context, email string) (int64, error) // INCR; if ==1 EXPIRE 900s
func (a *Auth) ResetLoginAttempts(ctx context.Context, email string) error               // DEL
func (a *Auth) GetLoginAttempts(ctx context.Context, email string) (int, error)          // GET, missing → 0
```

Match source semantics precisely:
- `IncrementLoginAttempts`: `INCR key`; **only when the result is 1**, set `EXPIRE key 900`. Return the count.
- `GetLoginAttempts`: `GET key`; nil/missing → `0`; parse int, unparseable → `0`.
- Key strings verbatim: `"blacklist:" + token`, `"login:attempts:" + email`.

Keep the raw `New` (client+ping) function as-is. Do not change boot flow.

### Step 3 — Audit-log insert query + regenerate

### Step 4 — JWT token service (`internal/module/auth/token.go`)

A small `TokenService` built from config (no DB/Redis). Use `github.com/golang-jwt/jwt/v5`, HS256.

```go
type TokenService struct {
    accessSecret  []byte
    refreshSecret []byte
    accessTTL     time.Duration
}

func NewTokenService(cfg *config.Config) *TokenService
```

Methods:
- `SignAccessToken(userID uuid.UUID, email string) (string, error)` — claims `sub=userID`, `exp=now+accessTTL`, `iat=now`, and **`email` only when non-empty**. (Register/login pass the email; refresh passes `""` → access token carries `sub` only, per the contract note.)
- `SignRefreshToken(userID uuid.UUID) (string, error)` — claims `sub=userID`, `iat=now`, `jti=uuid.NewString()`. **No `exp`.**
- `VerifyRefreshToken(token string) (uuid.UUID, error)` — parse with `refreshSecret`, enforce HS256 (`jwt.WithValidMethods([]string{"HS256"})`), return the `sub` as `uuid.UUID`. On any parse/signature failure return an error the handler maps to 401 `Invalid refresh token`.

Use `jwt.RegisteredClaims` (`Subject`, `IssuedAt`, `ExpiresAt`, `ID`) plus a custom struct embedding it with an `Email string \`json:"email,omitempty"\`` field. Secrets come from `cfg.JWTAccessSecret` / `cfg.JWTRefreshSecret`; `accessTTL = cfg.JWTAccessExpiresIn`.

func BindAndValidate(c echo.Context, req any) error {
    if err := c.Bind(req); err != nil {
        return echo.NewHTTPError(http.StatusBadRequest, "Invalid request body")   // parse → 400

### Step 8 — Auth service (`internal/module/auth/service.go`)

Port `service.ts` exactly. Define a **narrow** store interface so unit tests hand-mock only what's used (avoid mocking the whole `db.Querier`):

```go
type authStore interface {
    GetUserByEmail(ctx context.Context, email string) (db.User, error)
    CreateUser(ctx context.Context, arg db.CreateUserParams) (db.User, error)
    GetSessionByRefreshToken(ctx context.Context, refreshToken string) (db.Session, error)
    CreateSession(ctx context.Context, arg db.CreateSessionParams) (db.Session, error)
    RevokeSessionByID(ctx context.Context, id uuid.UUID) error
    RevokeSessionFamily(ctx context.Context, family uuid.UUID) error
    RevokeAllUserSessions(ctx context.Context, userID uuid.UUID) error
    WithTx(ctx context.Context, fn func(*db.Queries) error) error
}
```
`*database.Store` satisfies this (it embeds `*db.Queries` and has `WithTx`). Service deps: `store authStore`, `redisAuth *redis.Auth`, `audit *auditlog.Service`.

Methods (return `apperror.New(code)` on the documented failures; distinguish "no rows" via `errors.Is(err, pgx.ErrNoRows)`):

- **`Register(ctx, email, passwordHash, displayName *string) (db.User, error)`**
  1. `GetUserByEmail`; if a row is found → `apperror.New(apperror.EmailTaken)`. (`pgx.ErrNoRows` = OK, proceed.)
  2. `CreateUser`.
  3. `audit.Record(ctx, ActionUserRegister, &user.ID, nil, nil)` (best-effort).
  4. return user. *(Password hashing happens in the handler before calling — see Step 9 — matching where the source hashes; either place is fine, but keep it consistent and truncate to 72 bytes.)*

- **`Login(ctx, email, password string) (db.User, error)`**
  1. `attempts, _ := redisAuth.GetLoginAttempts(ctx, email)`; if `attempts >= 5` → `apperror.New(apperror.TooManyAttempts)`. **(check BEFORE credential validation).**
  2. `GetUserByEmail`. If not found **or** `bcrypt.CompareHashAndPassword` fails → `redisAuth.IncrementLoginAttempts(ctx, email)` then `apperror.New(apperror.InvalidCredentials)`. (Truncate password to 72 bytes before compare.)
  3. On success: `redisAuth.ResetLoginAttempts(ctx, email)`.
  4. `audit.Record(ctx, ActionUserLogin, &user.ID, nil, nil)`.
  5. return user.

- **`RotateSession(ctx, oldRefreshToken, newRefreshToken string, expiresAt time.Time) (uuid.UUID, error)`**:
  1. `GetSessionByRefreshToken(oldRefreshToken)` — **outside any transaction**; not found → `apperror.New(apperror.InvalidRefreshToken)`.
  2. If `session.IsRevoked` → `store.RevokeSessionFamily(session.Family)` (standalone call, **not** inside a `WithTx`) then `apperror.New(apperror.RefreshTokenReuse)`. **(reuse detection — revoke the whole family.)**
  3. If `session.ExpiresAt.Before(time.Now())` → `apperror.New(apperror.RefreshTokenExpired)`.
  4. Only now, wrap the two-write happy path in `store.WithTx(ctx, func(q *db.Queries) error { ... })`: `RevokeSessionByID(session.ID)`; `CreateSession{UserID: session.UserID, RefreshToken: newRefreshToken, Family: session.Family, ExpiresAt: expiresAt}`.
  5. return `session.UserID`.
  > **Do NOT wrap steps 1–3 in `WithTx`.** `Store.WithTx` rolls back the transaction whenever its callback returns a non-nil error (commits only on `nil`). If the reuse-detection `RevokeSessionFamily` call and the `return apperror.RefreshTokenReuse` live inside the same `WithTx` callback, the rollback silently undoes the family revocation — reuse detection would appear to work (401 returned) but never actually persist, so a subsequent refresh with a family-mate token would wrongly succeed. Caught live during Phase 2 execution: refreshing a revoked token's family-mate returned 200 instead of 401 until this was fixed. `RevokeSessionFamily` is a single `UPDATE`, already atomic on its own — it needs no transaction. Only the true multi-step write (revoke-old + create-new) goes in `WithTx`, and that callback only ever returns `nil` or a genuine DB error, never a business `apperror` — so `WithTx`'s commit-on-nil-only contract stays correct. This also matches the source more closely: `rotateSession` in `service.ts` has no explicit transaction at all.

- **`CreateSession(ctx, userID uuid.UUID, refreshToken string, family uuid.UUID, expiresAt time.Time) error`** — thin wrapper over `store.CreateSession` (used by register/login).

- **`RevokeAllSessions(ctx, userID uuid.UUID) error`** — `store.RevokeAllUserSessions(userID)` (already filters `is_revoked=false`).

### Step 9 — Auth handler (`internal/module/auth/handler.go`)

Mirror `index.ts`. Deps: `service *Service`, `token *TokenService`, `store` (for `GetSessionByRefreshToken` in logout), `redisAuth *redis.Auth`, `refreshTTL time.Duration` (= `cfg.JWTRefreshExpiresIn`). `Register(g *echo.Group)` mounts the four routes under the `/auth` group created in `server.New`.

- **`POST /auth/register`** (public):
  1. `BindAndValidate` → `RegisterRequest`.
  2. `hash := bcrypt.GenerateFromPassword(truncate72(pw), 12)`.
  3. `user, err := service.Register(ctx, email, string(hash), displayName)` → propagate err.
  4. `access := token.SignAccessToken(user.ID, user.Email)`; `refresh := token.SignRefreshToken(user.ID)`; `family := uuid.New()`.
  5. `service.CreateSession(ctx, user.ID, refresh, family, now+refreshTTL)`.
  6. `200 { accessToken, refreshToken }`.

- **`POST /auth/login`** (public): same as register from step 4 on, but call `service.Login` (which does rate-limit + bcrypt compare). Access token includes `user.Email`.

- **`POST /auth/refresh`** (public):
  1. `BindAndValidate` → `RefreshRequest`.
  2. `userID, err := token.VerifyRefreshToken(req.RefreshToken)`; err → `echo.NewHTTPError(401, "Invalid refresh token")` (matches source `status(401,'Invalid refresh token')`).
  3. `newRefresh := token.SignRefreshToken(userID)`.
  4. `uid, err := service.RotateSession(ctx, req.RefreshToken, newRefresh, now+refreshTTL)` → propagate err (maps to `INVALID_REFRESH_TOKEN` / `REFRESH_TOKEN_REUSE` / `REFRESH_TOKEN_EXPIRED`).
  5. `access := token.SignAccessToken(uid, "")` — **`sub` only, no email** (contract).
  6. `200 { accessToken: access, refreshToken: newRefresh }`.

- **`POST /auth/logout`** (public, reads Authorization if present):
  1. `BindAndValidate` → `RefreshRequest`.
  2. If `Authorization: Bearer <t>` present → `redisAuth.BlacklistToken(ctx, t, 15*time.Minute)`.
  3. `session, err := store.GetSessionByRefreshToken(ctx, req.RefreshToken)`; if found → `service.RevokeAllSessions(ctx, session.UserID)`. (Not found → ignore; still return success.)
  4. `200 { success: true }`. **Always succeeds**, even for an unknown/expired refresh token (parity).

Use `time.Now()` once per request for consistency. Strip the `"Bearer "` prefix exactly like the source (`strings.TrimPrefix`).

## Guardrails / notes for the executor

- **Parity is the contract.** The exact status codes and message strings in `docs/02` are non-negotiable — assert them literally in tests.
- **The reuse-detection + rate-limit ordering is the crux.** Rate-limit check happens *before* credential validation; reuse revokes the *whole family*; expiry is checked *after* the revoked check. Follow `service.ts` line-for-line.
- **Audit writes are best-effort** — a failed insert logs and is swallowed; it must never turn a 200 into a 500.
- **`sessions.expires_at` is `timestamp` without time zone — always use `time.Now().UTC()`, never bare `time.Now()`, for anything that writes to or compares against it.** pgx scans a "timestamp without time zone" column's raw digits as a UTC-tagged `time.Time` on read, but on write it stores whatever wall-clock digits the given `time.Time` carries, with **no conversion** — so a value built from a non-UTC local `time.Now()` gets its local digits stored verbatim, and reading it back (now UTC-tagged) shifts the apparent instant by the server's UTC offset. Caught live during Phase 2 execution on a UTC+7 dev host: an intentionally-expired test fixture (`time.Now().Add(-1h)`) read back ~7h in the future and `RotateSession`'s expiry check wrongly passed. Fixed at all three call sites: the two `expiresAt` computations in `handler.go` (`issueTokenPair`, `refresh`) and the comparison in `service.go`'s `RotateSession` — all now call `time.Now().UTC()`. Any future code that writes or reads `sessions.expires_at` (or any other `timestamp`-without-tz column) must follow the same convention.
- **Stay in the auth lane.** No middleware guards, no org/RBAC/subscription, no `GET /audit-logs`. If a step seems to need them, it's Phase 3/4 — stop and note it.
